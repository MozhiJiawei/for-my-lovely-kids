# 心愿管理 Use Cases

## 背景

心愿是孩子用小红花兑换奖励的地方。首版本先支持最简单的心愿内容：标题和需要的小红花数量；后续可以继续补充图片、链接、描述等更丰富的信息。

## UC1：录入心愿

- 家长在“心愿”页签中新增心愿。
- 新增心愿在独立编辑页面完成，方便后续承载图片、链接、描述等更丰富的信息。
- 家长填写心愿标题和需要的小红花数量。
- 家长需要选择心愿类型：
  - 一次性：实现后从当前心愿中退出。
  - 可重复：实现后仍然保留，之后还可以继续兑换。
- 家长可以选择是否让心愿优先出现在花园首页的心愿泡泡中。

## UC2：查看心愿

- 孩子在花园首页看到心愿泡泡。
- 置顶心愿优先展示；没有足够置顶心愿时，系统自动补充更适合展示的心愿。
- 首页心愿泡泡只用于查看和进入详情，不直接兑换，避免误触。
- 孩子点击泡泡进入心愿详情页。
- 心愿详情页展示心愿标题、需要的小红花、完成进度、心愿类型和兑换记录。

## UC3：兑换心愿

- 孩子可以在心愿列表页兑换心愿。
- 孩子可以在心愿详情页兑换心愿。
- 小红花足够时，兑换成功并扣减可用小红花。
- 兑换不会减少累计小红花。
- 小红花不足时，系统提示不能兑换。
- 一次性心愿只能实现一次。
- 可重复心愿可以多次实现，每次实现都会留下记录。
- 已实现的一次性心愿不再出现在当前心愿和首页泡泡中。

## UC4：管理当前心愿

- 心愿列表页展示当前仍可使用的心愿。
- 家长可以编辑心愿标题、所需小红花、心愿类型和首页优先展示设置。
- 编辑心愿时使用独立页面，方便后续增加图片、链接、描述等更多内容。

## UC5：查看历史兑换

- 心愿管理页展示历史兑换记录。
- 每条记录展示兑换时的心愿名称、小红花数量、状态和时间。
- 家长和孩子可以用历史记录回顾已经实现过的心愿。

## 验收口径

- 家长能新增、编辑心愿。
- 孩子能从首页泡泡进入详情，不能在首页直接兑换。
- 孩子能在列表页和详情页兑换心愿。
- 一次性和可重复心愿的表现符合各自规则。
- 支持多用户并发访问。
- 要求页面信息要准确。
