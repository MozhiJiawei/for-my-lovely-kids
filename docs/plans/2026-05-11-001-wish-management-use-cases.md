# 心愿管理 Use Cases

## 背景

心愿是孩子使用小红花兑换奖励的载体。首版本先保留极简描述能力：标题 + 需要的小红花数量；图片、富文本、链接等描述作为后续增强项。

## UC1：录入心愿

- 家长可以新增心愿，填写标题、所需小红花数量。
- 家长需要选择心愿类型：
  - `一次性`：兑换成功后自动归档，不再出现在当前心愿列表和首页泡泡。
  - `可重复`：兑换成功后仍保持可兑换，适合周期性奖励。
- 家长可以配置是否首页置顶。置顶心愿优先以首页心愿泡泡展示。

## UC2：查看与兑换心愿

- 首页只展示心愿泡泡，不直接兑换。
- 点击首页心愿泡泡进入心愿详情页。
- 孩子可以在心愿列表页兑换当前心愿。
- 孩子可以在心愿详情页兑换当前心愿。
- 兑换会扣减可用小红花，不影响累计小红花。
- 可用小红花不足时，兑换失败且不持久化部分兑换状态。

## UC3：管理当前心愿

- 心愿列表页展示当前可用心愿，包括标题、类型、所需小红花、置顶状态。
- 家长可以编辑心愿标题、所需小红花、类型、置顶状态。
- 首页展示规则：
  - 若存在置顶心愿，首页泡泡展示置顶心愿。
  - 若没有置顶心愿，按算法选择：一次性心愿优先于可重复心愿；同类型下高小红花优先于低小红花。

## UC4：查看历史兑换

- 心愿管理页展示历史兑换记录。
- 记录包含兑换时的标题快照、小红花快照、状态和时间。
- 后续可在此基础上增加图片、链接、备注、家长审批流等增强能力。

## 持久化验收

- 心愿新增、编辑、兑换都必须通过后端 API 写入数据库。
- API 测试覆盖新增和编辑心愿的持久化字段：类型、置顶、标题、小红花数量、状态。
- 兑换测试覆盖兑换记录、红花余额、流水写入，以及失败时不写入部分状态。
